# 自行车视觉理解与热力图系统重构架构图

## 1. 五层架构

```text
+--------------------------------------------------------------------------------------+
| 5. Generative Rendering Layer                                                         |
| AI 风格化、品牌视觉包装、海报图、报告图、首页英雄图                                   |
| 输入：标准 SVG、热力图图层、指标数据、品牌语义                                        |
| 输出：风格化 PNG、宣传图、封面图、生成式概念图                                        |
+--------------------------------------------------------------------------------------+
                                         ^
                                         |
+--------------------------------------------------------------------------------------+
| 4. Visualization Layer                                                                |
| 真实图热力图、标准 SVG 热力图、Exploded Heatmap、PNG/GIF/3D 图                        |
| 输入：标准部件槽位、颜色映射、指标值、识别置信度                                       |
| 输出：SVG/PNG/HTML/GIF/前端交互数据                                                   |
+--------------------------------------------------------------------------------------+
                                         ^
                                         |
+--------------------------------------------------------------------------------------+
| 3. Semantic Binding Layer                                                             |
| part_taxonomy -> component_catalog -> offer_snapshot / review_target / heatmap_metrics|
| 输入：视觉部件标签、部件名称、品牌提示、车型上下文                                     |
| 输出：结构化部件对象、价格/质量/性价比指标                                             |
+--------------------------------------------------------------------------------------+
                                         ^
                                         |
+--------------------------------------------------------------------------------------+
| 2. Canonicalization Layer                                                             |
| 自行车主体标准化、侧视模板对齐、部件槽位映射、SVG 基准模板                             |
| 输入：自行车 mask、关键几何点、部件掩膜                                                 |
| 输出：标准侧视坐标系、标准 SVG 槽位、Exploded 布局                                     |
+--------------------------------------------------------------------------------------+
                                         ^
                                         |
+--------------------------------------------------------------------------------------+
| 1. Perception Layer                                                                   |
| 自行车检测、主体分割、侧视判断、部件识别、部件分割                                     |
| 输入：原始图片/视频、白底商品图、用户上传图                                             |
| 输出：bike_mask、parts masks、view_score、confidence                                   |
+--------------------------------------------------------------------------------------+
```

## 2. 数据流

```text
用户图片 / 官网白底图
    -> 自行车检测
    -> 自行车主体提取
    -> 侧视可用性判断
    -> 部件识别与掩膜提取
    -> 映射到标准部件槽位
    -> 绑定数据库价格 / 评价 / 价值分数
    -> 生成 SVG / PNG / GIF / 3D 可视化
    -> 可选 AI 生图做风格化包装
```

## 3. 当前仓库中的对应模块

- 第一层：`annotation/`、`silhouette.py`、后续的 YOLO 推理脚本
- 第二层：标准 SVG 模板、`exploded_heatmap.py`
- 第三层：`unified_warehouse.py`、`website_api.py`
- 第四层：`generate_exploded_heatmap_assets.py`、`analyze_visualization_assets.py`
- 第五层：后续可接 `gpt-image` / `nanobanana` 等风格化 API

## 4. 重构价值

- 让视觉识别和渲染解耦
- 让数据库成为事实层和解释层
- 让 SVG 成为统一对比表达，而不是每张图独立上色
- 让 AI 生图只负责表现，不负责事实判断

## 5. 最小可运行闭环

```text
website_bikes_api.json
    -> 选定一辆车
    -> 转为标准 exploded heatmap payload
    -> 生成 exploded_heatmap.svg / png
```

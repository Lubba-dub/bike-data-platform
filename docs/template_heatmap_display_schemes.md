# 模板热力图显示方案与创新方向

## 1. 当前可落地前提

- 数据库导出规模
  - `website_bikes_api`: 71
  - `website_parts_api`: 115
  - `website_components_api`: 208
  - `reviews`: 125
- 当前模板已稳定挂接部件
  - `frame`
  - `fork`
  - `saddle`
  - `seatpost`
  - `handlebar`
  - `shiftlever`
  - `brake`
  - `tire`
- 当前模板新增但仍待继续补图形的部件
  - `pedal`
- 当前仍难以精细拆开的部件
  - `chainwheel`
  - `rearderailleur`
  - `freewhile`
  - `crank`

因此展示策略不应假装“所有部件都已被模板准确拆解”，而应采用三层表达：

1. 已映射部件直接着色
2. 有数据但未拆开的部件保留中性灰
3. 右侧卡片继续完整展示这些未映射部件的数据

## 2. 页面主方案

### 方案 A：标准 SVG 热力图主视图

最适合作为课程项目主页和最终答辩主画面。

- 中央使用标准侧视 SVG
- 顶部切换三种模式
  - `price_score`
  - `quality_score`
  - `value_score`
- 左右结构建议
  - 中部大画布展示自行车模板
  - 右侧展示部件详情卡和部件列表
- 交互建议
  - 悬停高亮模板部件
  - 点击部件切换详情卡
  - 点击模式时颜色平滑过渡

适合原因：

- 模板稳定，适合不同车型统一比较
- 不依赖真实照片视角完全一致
- 最能体现“数据库 + 模板 + 可视化”的课程主线

### 方案 B：真实图 / 剪影 / 模板三联视图

适合作为“视觉理解流程展示页”。

- 第一列：原始自行车图片
- 第二列：剪影或识别结果覆盖图
- 第三列：标准模板热力图

联动方式：

- 鼠标悬停任一部件时三列同步高亮
- 如果真实图中该部件识别置信度较低，可降低边框透明度
- 模板未拆出的部件只在右侧信息区显示，不强行在模板中制造虚假区域

适合原因：

- 更容易解释“为什么最终要落到标准模板”
- 能展示你项目里的视觉识别、标准化和可视化三步链路

### 方案 C：品牌-部件矩阵墙

适合作为数据库总览页。

- 行：品牌
- 列：部件类别
- 单元格颜色：平均价格、平均质量或数据覆盖率
- 单元格大小或边框强度：报价数 / 评论数

推荐切换维度：

- 平均价格分
- 平均质量分
- 平均性价比分
- 数据完整度

适合原因：

- 当前库的结构化数据已经足够支撑
- 不依赖模板拆分精度
- 能从“单车”扩展到“品牌与品类比较”

### 方案 D：部件卡片轮播与动态过渡

适合作为首页 hero 动效、GIF 或短视频展示。

- 固定一辆标准模板车
- 自动轮播 `price -> quality -> value`
- 切换时只有颜色和标题变化
- 下方同时轮播当前最突出的部件

适合原因：

- 成本低
- 视觉高级感容易做出来
- 很适合课程汇报开场页

## 3. 部件维度显示策略

### 3.1 价格热力图

目标：快速看出“哪里贵”。

建议：

- 使用暖色梯度
  - 浅灰白 -> 橙 -> 深红
- 高价部件优先突出
  - `frame`
  - `fork`
  - `tire`
- 对极端高值使用截断或对数归一化，避免 `frame` 完全压扁其他部件

### 3.2 质量热力图

目标：快速看出“哪里评价更强”。

建议：

- 使用冷色梯度
  - 浅灰白 -> 蓝 -> 深蓝
- 当 `reviews_count` 太少时降低饱和度
- 把“评分强，但评价样本少”的部件标注为低置信表达

### 3.3 性价比热力图

目标：看出“哪里更值”。

建议：

- 使用青绿梯度或蓝绿梯度
- 对 `value_score` 缺失部件保持中性灰
- 在详情卡中额外解释：
  - 是否因为缺价格
  - 是否因为缺评分
  - 是否因为模板未映射

### 3.4 数据完整度辅助层

这是很值得加入的一层创新表达。

- 填充颜色表达价格 / 质量 / 性价比
- 描边透明度表达数据完整度
- 微弱发光表达评论密度或报价密度

这样可以避免用户误把“灰色”理解成“差”，而是明确区分：

- 灰色可能是未映射
- 浅色可能是低值
- 透明可能是数据不足

## 4. 模板受限条件下的前端表达策略

当前最关键的不是继续强拆细部件，而是让界面表达真实、克制、可解释。

推荐规则：

- 已映射部件
  - 可点击
  - 可悬停
  - 可高亮
  - 直接着色
- 有数据但未映射部件
  - 不在模板中伪造点击区
  - 在部件列表和详情卡中保留
  - 用说明文案提示“数据库已有，模板尚未拆开”
- 完全缺数据部件
  - 保持中性灰
  - 卡片显示 `N/A`

对当前项目而言，这种“诚实表达”比强行细分更高级。

## 5. 创新方向

### 创新 1：双通道热力图

不要只表达一个数值。

- 填充色表达主指标
- 描边强度表达数据可信度

这样可以在同一张车上同时看见“贵不贵”和“证据够不够”。

### 创新 2：模板绑定的可解释视觉理解

把项目从“图像识别 demo”提升为“可解释视觉理解系统”。

核心链路：

1. 图片识别出部件
2. 识别结果绑定到标准模板槽位
3. 数据库指标绑定到模板槽位
4. 所有可视化都基于统一槽位渲染

这样未来无论是 SVG、PNG、GIF 还是 3D 展示，都共用同一套部件对象。

### 创新 3：故事化模式切换

不仅是切换 tab，而是让页面讲故事。

例如：

- 第一步：显示价格热力图，说明整车成本结构
- 第二步：切到质量热力图，说明哪些部件评价更稳定
- 第三步：切到性价比热力图，说明哪些部件最值得升级

这比静态图更适合答辩、演示视频和网页首页。

### 创新 4：升级建议层

在现有数据库基础上增加一个很有价值的功能：

- 对当前车的各部件按“低性价比、高价格、低评分”排序
- 自动给出“建议优先升级部件”

例如：

- 先升级轮组
- 再升级制动
- 最后升级传动

这会让你的系统从“展示”走向“决策支持”。

### 创新 5：3D Exploded Stage

后续数据库更完整后，可把当前模板进一步扩展到 3D 舞台：

- 中间是侧视标准车
- 部件沿 z 轴轻微分离
- 部件颜色仍由数据库指标决定
- 滚动页面时完成分解与回收动画

这类表达适合 Three.js，适合作为高端展示版首页。

## 6. 实施优先级

### 第一阶段：立即可交付

- 标准 SVG 热力图单页
- 模式切换
- 部件详情卡
- 已映射 / 未映射部件区分
- Inkscape 导出 PNG 预览

### 第二阶段：适合课程汇报增强

- 真实图 / 剪影 / 模板三联视图
- 品牌-部件矩阵墙
- 动态模式轮播 GIF
- 数据完整度辅助层

### 第三阶段：创新展示版

- 升级建议层
- 3D 指标空间
- Three.js exploded stage
- 多车对比叙事页面

## 7. 结论

在当前模板只能有限补充、仅适合新增 `pedal` 的情况下，最优路线不是继续强拆，而是：

- 把现有模板做成高质量交互前端
- 用数据库保证事实完整
- 用中性灰和说明卡片诚实表达模板边界
- 再通过矩阵墙、故事化切换和升级建议层，把项目提升为一个“可解释的自行车部件数据可视化系统”

这条路线技术风险更低，展示效果更稳定，也更符合课程项目从数据到交互的完整闭环。
